## 二、核心模块技术选型

### 2.1 技术选型对比

| 维度 | 选型 | 备选方案 | 选型理由 |
|------|------|----------|----------|
| 前端框架 | 原生 HTML/CSS/JS | React / Vue | 零依赖、零构建、极简交付 |
| 后端运行时 | Node.js HTTP | Express / Koa / Python Flask | 零依赖、与前端同语言、流媒体原生支持 |
| 数据存储 | JSON 文件 | SQLite / MongoDB | MVP 阶段不需要数据库，JSON 可直接编辑 |
| AI 模型 | Doubao-Seed-2.0-lite | GPT-4o / Claude | 火山方舟 API，国内可用，性价比高 |
| 视频协议 | HTTP Range | HLS / DASH | MP4 文件直接服务，实现简单 |
| 弹幕实现 | CSS Animation | Canvas / WebGL | 轻量、性能好、GPU 加速 |
| 部署方案 | Node + Cloudflare Tunnel | Docker + Nginx | 个人 PC 直接部署，零成本公网访问 |

### 2.2 核心技术要点

**视频播放器**: 使用原生 `<video>` 元素，通过 CSS 隐藏原生控件，自行实现播控 UI。支持 HTTP Range 请求实现拖动进度。全屏使用 Fullscreen API。

**弹幕引擎**: 基于 CSS `@keyframes` 实现从右到左的平滑滚动。每条弹幕创建独立 DOM 元素，`animationend` 事件自动回收。预置弹幕和用户弹幕通过不同 class 区分样式。

**高光触发**: 视频 `timeupdate` 事件驱动，每秒多次比对 `currentTime` 与高光 `startTime-endTime` 区间。防抖机制避免边界重复触发。

**AIGC 插入**: 在高光触发时暂停主视频，显示 AIGC 生成的内容覆盖层。倒计时结束后自动恢复播放。支持跳过按钮。

### 3.2 AI 高光识别流程

```
剧集 MP4 文件
    │
    ▼
analyze_episode.py
    │
    ├── 读取 model-settings.json（API Key + Endpoint ID）
    ├── 读取 Prompt 模板
    ├── 填充模板变量（剧名、剧集名）
    │
    ▼
调用 Doubao-Seed-2.0-lite API
    │
    ▼
模型返回高光 JSON
    │
    ├── 解析 JSON（strip_json_fence / extract_json_candidate）
    ├── 归一化字段（normalizeImportedHighlight）
    ├── 生成唯一 ID
    │
    ▼
导入后端 (POST /api/model/import)
    │
    ▼
存储到 store.json / db.json
    │
    ▼
审核台审核 → 发布 (draft → published)
    │
    ▼
播放器可触发互动
```

## 五、AI 参与部分说明

本项目允许并鼓励在各环节使用 AI 辅助，具体 AI 参与部分如下：

### 5.1 AI 辅助编码

- **后端 API 实现**: Claude Code 辅助编写 server.js 中的路由、数据处理、模型代理逻辑
- **前端交互逻辑**: AI 辅助实现弹幕引擎、动效系统、互动组件、分支交互
- **CSS 样式**: AI 辅助编写动效动画、响应式布局、弹幕动画
- **批量脚本**: AI 辅助编写 batch_analyze.py 批量分析脚本

### 5.2 AI 模型调用

- **高光识别**: 调用 Doubao-Seed-2.0-lite 多模态能力分析短剧视频
- **剧情续写**: 调用大模型根据剧尾钩子生成多个分支剧本
- **互动文案生成**: 调用大模型优化互动按钮文案
- **视频分支构思**: 调用大模型根据高光点头脑风暴视频分支创意

- **README**: AI 辅助生成项目 README
